Durante toda a descrição do cenário que será utilizado no software, o Domain Expert começa a utilizar termos do próprio negocio, porem quando o contexto (departamentos de uma empresa) um outro termo pode ser utilizado para o mesmo termo utilizado anteriormente.

Para isso é super importante a definição universal do negocio e que posteriormente será utilizado dentro do desenho do sistema.

Essa linguagem deve ser consistente e detalhada, para facilitar o entendimento e a comunicação.

## Termos Ambíguos
Dentro de um mesmo subdomínio pode acontecer de ter um termo ter mais de um significado, na criação da linguagem ubíqua precisamos definir a definição para cada termo, definindo todos os significados para o mesmo termo.

## Termos Sinônimos
Sempre devemos tentar quebrar esse termos, são termos que são utilizados por vários significados.
